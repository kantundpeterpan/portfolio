Computing in j (III)

In the previous exercise, you were able to pass expressions in j that returned a vector. Recall that if you want to return a data.table instead, wrap the expression with list() (or it’s alias, .()). This lets you:

  • Rename the column result
  • Compute more than one expression on a column
  • Compute on more than one column

Computations by Groups in data.table

This lesson covers using the by argument in data.table to perform grouped computations.

The by Argument

  • Allows grouping data and performing computations for each group

  • Basic syntax: DT[, j, by=...]

  • Example:

    # Total trips by start station
DT[, .N, by=start_station]

  • by can take a character vector or list of expressions

  • Grouping column is always returned first

  • Can rename columns on the fly:

    DT[, .(no_trips = .N), by=.(start = start_station)]

Expressions in by

  • Can compute new columns for grouping:

    DT[, .N, by=.(start_station, 
              mon = month(start_date))]

  • Allows complex grouping in a single line of code

Key Points

  • .N special symbol gives count of rows
  • by=.() syntax allows renaming and computing columns
  • Grouping column always returned first
  • Unnamed computations get automatic names (e.g. “N” for .N)

References

  • data.table documentation: https://rdatatable.gitlab.io/data.table/
  • R for Data Science: https://r4ds.had.co.nz/

Chaining data.table Expressions

  • Chaining allows performing successive operations on outputs without intermediate assignments

  • Example: Finding three shortest trips over 1 hour

    batrips[duration > 3600][order(duration)][1:3]

  • Advanced example: Top three start stations with lowest mean duration

    batrips[, .(mean_duration = mean(duration)), by = start_station][order(mean_duration)][1:3]

uniqueN() Function

  • Returns count of unique elements in an object

  • Works on vectors, data frames, and data tables

  • Usage:

    uniqueN(vector)
uniqueN(data_table)
uniqueN(data_table, by = "column_name")

Combining uniqueN() with by

  • Useful for grouped operations

  • Example: Number of unique bike IDs for each month

    batrips[, .(unique_bikes = uniqueN(bike_id)), by = .(month = month(start_date))]

Key Takeaways

  1. Chaining expressions improves code readability and efficiency
  2. uniqueN() is a versatile function for counting unique elements
  3. Combining uniqueN() with by allows for powerful grouped operations

Computations in R using .SD and data.table

Introduction to .SD

  • .SD stands for “Subset of Data”
  • Powerful symbol for complex data wrangling
  • When grouping, holds intermediate data for each group
  • Contains all columns except grouping column by default

Using .SD

  • .SD contains columns and rows for each group
  • Groups are data tables, allowing further operations

Examples:

First row for each group:

x[, .SD[1], by=id]

Last row for each group:

x[, .SD[.N], by=id]

.SDcols

  • Controls columns available to .SD
  • Takes character vector of column names
  • Can include or exclude specific columns

Examples:

Return specific columns:

DT[, .SD[1], by=start_station, .SDcols=c("trip_id", "duration")]

Exclude specific columns:

DT[, .SD[1], by=start_station, .SDcols=!c("end_station", "bike_id")]

Key Points

  • .SD is powerful for group-wise operations
  • .SDcols allows fine-grained control over included columns
  • Combine .SD with other data.table features for complex operations

Data Table Reference Semantics in R

Understanding Memory Management

Data Frame Behavior (Pre-R 3.1.0)

  • Deep copies entire data frame for column updates
  • Example: 10GB dataframe requires additional 10GB RAM for single value update
  • Highly inefficient memory usage

Data Frame Improvements (R 3.1.0+)

  • Only copies modified columns instead of entire dataframe
  • Still requires additional memory for column copies
  • Example: 10GB dataframe with 50/100 columns = 5GB extra space needed

data.table Reference Semantics

Key Features

  • No deep copying of objects
  • No temporary variables
  • Direct updates to original data table
  • Uses := operator for reference updates
  • No assignment needed for modifications

Syntax Options

  1. LHS := RHS Form
# Single column (quotes optional)
DT[, col1 := value]

# Multiple columns
DT[, c("col1", "col2") := list(value1, value2)]

# Example: Adding computed columns
batrips[, c("is_dur_gt_1hour", "weekday") := 
  list(duration > 3600, weekdays(start_date))]
  1. Functional Form
# Multiple operations
DT[, `:=`(
  col1 = value1,
  col2 = value2
)]

# Delete column (assign NULL)
DT[, `:=`(column_name = NULL)]

# Example: Update and delete
batrips[, `:=`(
  is_dur_gt_1hour = NULL,  # Delete column
  start_station = toupper(start_station)  # Update to uppercase
)]

Best Practices

  • Choose one syntax style and stick to it
  • Use reference semantics for memory efficiency
  • Remember updates are made in-place
  • No need to reassign results

Performance Benefits

  • Extremely fast operations
  • Minimal memory overhead
  • Ideal for large datasets
  • Efficient for multiple column modifications

Adding a new column

As mentioned in the video, data.table does not have to deep copy objects in order to modify them. It can modify objects by reference (inplace) to save memory and runtime using the := operator. Grouped Aggregations and Column Updates

Key Concepts

  • Using := operator for column updates by reference
  • Combining := with by for grouped operations
  • Chaining operations in data.table
  • Managing intermediate columns

Common Patterns

Adding Group Counts

# Count trips by zip code
batrips[, n_zip_code := .N, by = zip_code]

Filtering Based on Group Counts

# Filter zip codes with >1000 trips
batrips[, n_zip_code := .N, by = zip_code][n_zip_code > 1000]

Removing Intermediate Columns

# Complete operation with cleanup
batrips[, n_zip_code := .N, by = zip_code
       ][n_zip_code > 1000
       ][, n_zip_code := NULL]

Important Notes

  • Updates by reference are silent (no console output)
  • Use empty brackets [] to print updated data.table
  • Chain operations using multiple []
  • Use := NULL to remove columns

Best Practices

  • Create intermediate columns when needed for calculations
  • Clean up temporary columns in final result
  • Chain operations for efficient data manipulation
  • Use by reference operations for memory efficiency

Syntax Reference

  • := : Update by reference
  • by = : Group operations
  • .N : Number of rows in group
  • [, col := NULL] : Delete column

Advanced Data Aggregations with data.table

Key Concepts

The := Operator

  • Used for adding/updating columns by reference
  • Two forms:
    • LHS := RHS form
    • Functional form
  • Can be combined with by argument for grouped operations

Multiple Column Operations by Group

# Add multiple columns by reference for each group
DT[, `:=`(
  first_dur = first(duration),
  last_dur = last(duration)
), by = end_station]

Binning and Categorization

# Create categories based on median duration
DT[, trip_category := {
  med_dur = median(duration)
  if (med_dur < 500) "short"
  else if (med_dur < 1000) "medium"
  else "long"
}, by = .(start_station, end_station)]

Multi-line Expressions in j

Using Curly Brackets

# Complex multi-line operations
DT[, trip_category := {
  med_dur = median(duration)
  if (med_dur < 500) {
    "short"
  } else if (med_dur < 1000) {
    "medium"
  } else {
    "long"
  }
}, by = .(start_station, end_station)]

Using Custom Functions

# Alternative using function
bin_median_duration <- function(x) {
  med_dur = median(x)
  if (med_dur < 500) return("short")
  else if (med_dur < 1000) return("medium")
  else return("long")
}

DT[, trip_category := bin_median_duration(duration), 
   by = .(start_station, end_station)]

Advanced Operations

Combining i, j, and by

# Find minimum duration > 500 for each station combination
DT[duration > 500, 
   .(min_long_duration = min(duration)), 
   by = .(start_station, end_station)]

Key Points: - Filtering occurs first (i) - Then grouping is applied (by) - Finally, computation is performed (j) - Returns NA for groups with no qualifying rows

Best Practices

  • Use := for efficient in-place modifications
  • Wrap complex operations in curly brackets
  • Consider custom functions for reusable logic
  • Remember evaluation order: i → by → j

Fast Data Import with fread()

Key Features

Performance - Blazing fast parallel import using multi-core processing - Controls threads via nThread argument - Default: uses all available threads

Versatility - Reads from local files, URLs, and strings with same syntax - Auto-detects data source type - Auto-guesses column types - Handles quotes, separators, whitespace - Note: dates/datetimes read as character (convert using fasttime/anytime packages)

Usage Examples

Basic Import Syntax

# From file
dt <- fread("path/to/file.csv")

# From URL
dt <- fread("http://example.com/data.csv")

# From string
dt <- fread("a,b\n1,2\n3,4")

# Verbose mode
dt <- fread("data.csv", verbose=TRUE)

Control Parameters

Row Control - nrows: Specify number of rows to read (excluding header) - skip: Skip lines before parsing

# Read first 100 rows
dt <- fread("file.csv", nrows=100)

# Skip first 5 lines
dt <- fread("file.csv", skip=5)

# Skip until pattern match
dt <- fread("file.csv", skip="start_data")

# Combine skip and nrows
dt <- fread("file.csv", skip=5, nrows=100)

Column Selection - select: Import only specified columns - drop: Import all except specified columns - Cannot use both simultaneously

# Select specific columns
dt <- fread("file.csv", select=c("col1", "col3"))
# or by position
dt <- fread("file.csv", select=c(1,3))

# Drop columns
dt <- fread("file.csv", drop=c("col2", "col4"))
# or by position
dt <- fread("file.csv", drop=c(2,4))

Best Practices

  • Use verbose=TRUE for debugging
  • Convert date/time columns post-import
  • Test with small nrows for large files
  • Consider column selection for memory efficiency

Related Packages

  • fasttime: Fast datetime conversion
  • anytime: Flexible datetime parsing

Advanced Data Reading with fread() in data.table

Key Features Overview

Integer64 Type Handling

  • R integer limit: \[2^{31} - 1 = 2147483647\]
  • Default behavior: Uses bit64 package for large integers
  • Can override with numeric or character types using colClasses

Column Class Specification

Using colClasses argument in three ways:

  1. Named vector:
# Column names specified before parsing
fread("file.csv", colClasses = c(col1 = "numeric", col2 = "character"))
  1. Unnamed vector:
# Classes assigned sequentially
fread("file.csv", colClasses = c("numeric", "character", "integer"))
  1. Named list of vectors:
# Efficient for multiple columns of same type
fread("file.csv", colClasses = list(
  numeric = 1:4,                    # First four columns as numeric
  character = c("name", "address"), # Specific columns as character
  integer = 5:10                    # Columns 5-10 as integer
))

Handling Missing Data

Fill Argument

  • Default: fill = FALSE
  • Set fill = TRUE for incomplete columns
  • Behavior:
    • Empty strings for character type
    • NA for integer/logical/numeric types
# Example with fill
fread("incomplete.csv", fill = TRUE)

NA Strings

  • Use na.strings for custom NA values
  • Memory efficient parsing
  • Accepts character vector
# Example with custom NA values
fread("data.csv", na.strings = c("N/A", "missing", "NULL"))

Best Practices

  • Always specify colClasses for critical data types
  • Use fill = TRUE when dealing with ragged data
  • Define na.strings explicitly for consistent NA handling
  • Consider using integer64 for large integer values

Fast Data Writing with fwrite()

Overview

fwrite() is data.table’s high-performance parallel file writer, offering significant speed improvements over base R’s write.csv(). It utilizes multiple threads by default and provides intelligent defaults for most common use cases.

Key Features

Multi-threaded Performance - Uses all available threads by default - Thread count controllable via nThread parameter - Significantly faster than base R alternatives

Basic Usage

# Basic write
fwrite(DT, "output.csv")

# Control threads
fwrite(DT, "output.csv", nThread=4)

List Column Handling - Automatically flattens list columns - Uses “|” as default secondary separator - Customizable via sep2 parameter

DT = data.table(
  A = 1:3,
  B = list(1:2, 3:4, 5:6)
)
fwrite(DT, "list_cols.csv", sep2=c("{",",","}"))

DateTime Handling

ISO Format (Default) - International standard format - Prevents ambiguity - Fast processing

dt = data.table(
  date = as.IDate("2023-01-01"),
  time = as.ITime("10:30:00"),
  datetime = as.POSIXct("2023-01-01 10:30:00")
)
fwrite(dt, "dates_iso.csv")

Squash Format - Removes separators (-, :) - Returns integers - Useful for extraction operations

# Write with squashed format
